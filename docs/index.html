CopilotScope

Observability for GitHub Copilot chat sessions. Real-time quality scores, turn-level friction analysis, and token economics — powered by OpenTelemetry, zero external services required.

→ Get Started View on GitHub Research Paper

What it is

Measure every session.
Score it automatically.

CopilotScope captures GitHub Copilot chat sessions as OpenTelemetry spans and computes a composite quality score (0–100) in real time. No extra SDK, no data leaving your machine — just one OTLP endpoint pointed at the collector.

The six-component engine weighs reliability, acceptance, friction, latency, feedback, and efficiency. Only components with actual data enter the composite — no sessions are pinned near a neutral prior when telemetry is sparse.

Sibling project of AgentHelm: Scope observes (what was that session worth?), Helm steers (run the session, approve the tools, keep the record).

What we measure

Six lenses on every session

📊

Composite score 0–100

Six weighted components: reliability 0.25, acceptance 0.20, friction 0.20, latency 0.15, feedback 0.10, efficiency 0.10. Weights renormalize to data that actually arrived — no phantom penalty for missing telemetry.

🔬

Turn analysis (TFRA)

Every invoke_agent trace is one turn, scored for friction — LLM/tool errors, latency vs. this session's median TTFT, and repair loops (tool-call bursts with failures). Best and worst turns surfaced with reasons.

✂️

Edit survival analysis

Four-gram matching plus a no-revert split (0.4/0.6 weight). Measures whether edits the agent suggested actually survived downstream — the truest signal of acceptance quality.

💰

Token & cache economics

Per-model cost with configurable pricing (CopilotScope:Pricing), cache hit savings, cost per turn, and cost per accepted edit. Understand what each session actually spent.

⏱️

Latency-utility model

Per-sample utility curve with >2 s and >8 s risk buckets. Translates raw TTFT numbers into an actionable quality signal — slow responses hurt differently at different thresholds.

😤

Frustration detection

EN/PL lexicon, Jaccard-distance rephrasing detection, and typography signals (caps lock, repeated punctuation). Local heuristic today; cloud deep-classifier via the judge agent is planned.

🗄️

Postgres persistence

Write-behind upsert every second — telemetry bursts never cause write storms. Rehydrates sessions on startup. A Postgres outage degrades gracefully to in-memory; ingest never blocks.

📈

Overview dashboard

Cross-session summary: total input/output/cache tokens, tokens per day, calls per model, top sessions by token burn, and average quality. Everything you burned across all chats in one view.

Quality engine

A score that reflects real work, not a neutral prior

v2 of the quality engine renormalizes weights across components that have data. A session without feedback telemetry isn't penalized — the remaining five components absorb the weight and score what actually happened.

Algorithm status

Local-first, cloud-ready

Seven algorithms run entirely on your machine. Three requiring LLM access (G-Eval, SPUR, RAGAS) are planned for the cloud deployment via Azure AI Foundry — they register as "no-data" locally, never as an error.

Quick start

Up in under a minute

Prerequisites: .NET 8 SDK and Docker. No workloads — Aspire 9 ships as a NuGet MSBuild SDK.

# Pull from GHCR — no clone needed # Linux / macOS / Git Bash: curl -O https://raw.githubusercontent.com/konradcinkusz/copilotscope/master/docker-compose.ghcr.yml # Windows PowerShell: curl.exe -O https://raw.githubusercontent.com/konradcinkusz/copilotscope/master/docker-compose.ghcr.yml docker compose -f docker-compose.ghcr.yml up # Collector → http://localhost:4318 (OTLP ingest + REST API) # Dashboard → http://localhost:5200 (Blazor UI)

One-time setup: GHCR packages start private — switch each to public (GitHub → Packages → Settings) so docker pull works anonymously. Then point your Copilot client at http://localhost:4318.

# Clone and start with .NET Aspire git clone https://github.com/konradcinkusz/copilotscope cd copilotscope dotnet run --project src/CopilotScope.AppHost # Aspire starts: Postgres (container, named volume) # pgAdmin (browse the sessions table) # Collector → http://localhost:4318 # Dashboard → http://localhost:5200 # Generate demo telemetry without a real Copilot subscription: dotnet run --project tools/CopilotScope.TelemetryGen -- http://localhost:4318 my-session

F5 on CopilotScope.AppHost does the same with the debugger attached to everything. Run dotnet test to execute the full test suite (decoder, quality, persistence round-trips).

// VS Code settings.json — enable Copilot OTel { "github.copilot.chat.otel.enabled": true, "github.copilot.chat.otel.otlpEndpoint": "http://localhost:4318", "github.copilot.chat.otel.captureContent": true // captureContent stores prompt/response previews (last 100 entries, // truncated at 4 000 chars). Omit if you prefer metadata-only. } # Copilot CLI — one-liner setup (PowerShell): .\scripts\Enable-CopilotOtel.ps1 -CaptureContent copilot # run from the same terminal

Note: the CLI follows the OTel GenAI standard — OTEL_INSTRUMENTATION_GENAI_CAPTURE_MESSAGE_CONTENT=true — which the script sets. COPILOT_OTEL_CAPTURE_CONTENT is not a real variable.

Try it without a Copilot subscription

  1. Start the stack (Docker or Aspire)
  2. Run dotnet run --project tools/CopilotScope.TelemetryGen -- http://localhost:4318 demo
  3. Open the Dashboard — a session named demo appears with realistic quality scores
  4. Click the session to see turn analysis, score components, and the prompt transcript
